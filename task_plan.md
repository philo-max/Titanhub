# 任务规划：双管齐下（云端部署 + 本地 JS 引擎）

## 目标声明
实现移动端的自主独立使用：
1. **路线 A (云端部署)**：提供后端的一键部署配置，让移动端能直接通过公网 API 运行。
2. **路线 B (本地 JS 引擎)**：集成 Flutter 离线 JS 沙箱，能够在无服务器情况下解析插件并在本地 SQLite (Drift) 中持久化。

## 阶段划分
- [x] 阶段 1：本地 JS 沙箱技术验证与基础框架搭建
- [x] 阶段 2：开发 Dart-JS 虚拟机桥接逻辑（Dio 网络代理与 cheerio 选择器）
- [x] 阶段 3：重构移动端数据获取逻辑为本地/云端双通道混合模式
- [x] 阶段 4：配置云端部署生态（如 Docker/Vercel/CF Workers 一键配置）
- [ ] 阶段 5：全端自动化编译与双通道自主运行可用性测试
